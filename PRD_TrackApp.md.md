# وثيقة متطلبات المنتج (PRD)
## تطبيق: TrackMe — تتبع الدوام والمالية الشخصية

**الإصدار:** 1.0  
**التاريخ:** مايو 2026  
**الحالة:** مسودة أولية

---

## 1. نظرة عامة على المنتج

### 1.1 ملخص تنفيذي
TrackMe هو تطبيق ويب (Web App) محلي يعمل على الشبكة الداخلية، يُمكّن المستخدمين من تتبع ساعات عملهم ورواتبهم (أسبوعية وشهرية)، وإدارة مصاريفهم اليومية، ووضع أهداف مالية ومتابعة تحقيقها.

### 1.2 المشكلة التي يحلها
- صعوبة تتبع أيام وساعات العمل الفعلية ومقارنتها بالراتب المستحق.
- غياب أداة مركزية لربط الدخل بالمصاريف والأهداف المالية.
- الحاجة لحل محلي كامل لا يعتمد على خدمات سحابية مدفوعة.

### 1.3 الهدف العام
بناء تطبيق ويب بسيط وسريع، يعمل محلياً، يدعم أكثر من مستخدم، ويوفر رؤية مالية واضحة وشاملة لكل مستخدم.

## 2. المستخدمون المستهدفون

| الفئة | الوصف |
|-------|-------|
| المستخدم الأساسي | أفراد يعملون بنظام الراتب الشهري أو الأسبوعي ويريدون تتبع دوامهم وماليتهم |
| مستخدمون إضافيون | أشخاص آخرون على نفس الشبكة المحلية يستخدمون حساباتهم المستقلة |

## 3. المتطلبات الوظيفية

### 3.1 نظام المصادقة (Authentication)

| # | المتطلب |
|---|---------|
| AUTH-01 | تسجيل مستخدم جديد (اسم، بريد أو اسم مستخدم، كلمة مرور) |
| AUTH-02 | تسجيل الدخول وتسجيل الخروج |
| AUTH-03 | كل مستخدم يرى بياناته الخاصة فقط — لا مشاركة بين المستخدمين |
| AUTH-04 | حماية كلمة المرور بالتشفير (hashing) |

---

### 3.2 وحدة تتبع الدوام

#### 3.2.1 إدخال بيانات الدوام

| # | المتطلب |
|---|---------|
| ATT-01 | إدخال يدوي: تاريخ اليوم، وقت البداية، وقت النهاية |
| ATT-02 | حساب تلقائي لعدد ساعات العمل في اليوم |
| ATT-03 | دعم تسجيل أيام غياب أو إجازة |
| ATT-04 | تعديل أو حذف سجل دوام مُدخَل سابقاً |

#### 3.2.2 الراتب والحسابات

| # | المتطلب |
|---|---------|
| SAL-01 | دعم نمطين للراتب: **شهري** و**أسبوعي** |
| SAL-02 | المستخدم يُدخل قيمة الراتب الشهري أو الأسبوعي عند الإعداد |
| SAL-03 | حساب الراتب المستحق بناءً على أيام العمل الفعلية مقارنةً بأيام العمل المتوقعة في الفترة |
| SAL-04 | عرض الفرق بين الراتب المستحق والراتب الكامل (إن وُجد غياب) |
| SAL-05 | العملة: دولار أمريكي ($) |

---

### 3.3 وحدة المصاريف

| # | المتطلب |
|---|---------|
| EXP-01 | إضافة مصروف جديد: التاريخ، المبلغ، الفئة، ملاحظة اختيارية |
| EXP-02 | فئات المصاريف الافتراضية: طعام، إيجار، مواصلات، ترفيه، صحة، أخرى |
| EXP-03 | إمكانية إضافة فئة مخصصة |
| EXP-04 | تعديل أو حذف مصروف مُسجَّل |
| EXP-05 | عرض إجمالي المصاريف لكل فترة (أسبوع / شهر) |

---

### 3.4 وحدة الأهداف المالية

| # | المتطلب |
|---|---------|
| GOAL-01 | إنشاء هدف مالي: اسم الهدف، المبلغ المستهدف، تاريخ الاستحقاق |
| GOAL-02 | ربط الهدف تلقائياً بالدخل المستحق والمصاريف لحساب المتاح للادخار |
| GOAL-03 | عرض شريط تقدم لكل هدف (نسبة التحقق) |
| GOAL-04 | تعديل أو حذف هدف |
| GOAL-05 | إشعار مرئي داخل التطبيق عند الاقتراب من تحقيق الهدف أو تجاوز الميزانية |

---

### 3.5 لوحة التحكم والتقارير (Dashboard)

| # | المتطلب |
|---|---------|
| DASH-01 | ملخص أسبوعي: ساعات العمل، الراتب المستحق، المصاريف، الصافي المتبقي |
| DASH-02 | ملخص شهري: نفس البيانات على مستوى الشهر |
| DASH-03 | مقارنة بصرية بين الدخل الكلي والمصاريف الكلية |
| DASH-04 | عرض تقدم الأهداف المالية النشطة |
| DASH-05 | فلترة التقارير بالأسبوع أو الشهر |

## 4. المتطلبات غير الوظيفية

| المتطلب | التفاصيل |
|---------|----------|
| النوع | Web App — يعمل عبر المتصفح على الشبكة المحلية |
| التخزين | محلي بالكامل — لا يُرسَل أي بيانات للإنترنت |
| قاعدة البيانات | مجانية — مقترح: **SQLite** (بسيطة، لا تحتاج خادم منفصل) أو **PostgreSQL** |
| المستخدمون المتزامنون | يدعم عدة مستخدمين على نفس الشبكة |
| اللغة | واجهة باللغة العربية (أو ثنائية اللغة عربي/إنجليزي) |
| الأجهزة المدعومة | متصفح ويب على كمبيوتر مكتبي أو لابتوب |
| الأمان | كل بيانات المستخدم معزولة تماماً عن بقية المستخدمين |

---

## 5. التقنيات المقترحة

| الطبقة | الخيار المقترح | السبب |
|--------|---------------|-------|
| الواجهة الأمامية | React.js أو Vue.js | سريع، مرن، مجتمع واسع |
| الواجهة الخلفية | Node.js (Express) أو Python (FastAPI) | خفيف وسهل النشر محلياً |
| قاعدة البيانات | SQLite | مجانية، لا تحتاج تثبيت إضافي، مثالية للاستخدام المحلي |
| المصادقة | JWT Tokens | آمنة وخفيفة |
| التثبيت | Docker (اختياري) | لتسهيل تشغيله على أي جهاز في الشبكة |

## 6. شاشات التطبيق الرئيسية

```
1. صفحة تسجيل الدخول / إنشاء حساب
2. لوحة التحكم (Dashboard) — الصفحة الرئيسية
3. صفحة الدوام — إدخال وعرض السجلات
4. صفحة المصاريف — إضافة وتتبع المصاريف
5. صفحة الأهداف المالية — إنشاء ومتابعة الأهداف
6. صفحة التقارير — ملخص أسبوعي وشهري
7. صفحة الإعدادات — إدخال بيانات الراتب وتفضيلات الحساب
```

## 7. نموذج البيانات (مبسّط)

### جدول المستخدمين (Users)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INTEGER | المعرّف الفريد |
| username | TEXT | اسم المستخدم |
| password_hash | TEXT | كلمة المرور المشفرة |
| salary_type | TEXT | monthly / weekly |
| salary_amount | REAL | قيمة الراتب بالدولار |

### جدول سجلات الدوام (Attendance)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INTEGER | المعرّف الفريد |
| user_id | INTEGER | مرتبط بالمستخدم |
| date | DATE | تاريخ اليوم |
| start_time | TIME | وقت البداية |
| end_time | TIME | وقت النهاية |
| hours_worked | REAL | ساعات العمل المحسوبة |
| status | TEXT | worked / absent / holiday |

### جدول المصاريف (Expenses)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INTEGER | المعرّف الفريد |
| user_id | INTEGER | مرتبط بالمستخدم |
| date | DATE | تاريخ المصروف |
| amount | REAL | المبلغ بالدولار |
| category | TEXT | فئة المصروف |
| note | TEXT | ملاحظة اختيارية |

### جدول الأهداف المالية (Goals)
| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | INTEGER | المعرّف الفريد |
| user_id | INTEGER | مرتبط بالمستخدم |
| name | TEXT | اسم الهدف |
| target_amount | REAL | المبلغ المستهدف |
| due_date | DATE | تاريخ الاستحقاق |
| saved_amount | REAL | المبلغ المدخّر حتى الآن |

## 9. معايير القبول (Definition of Done)

- يعمل التطبيق بالكامل على الشبكة المحلية بدون اتصال إنترنت.
- كل مستخدم لا يرى إلا بياناته الخاصة.
- حسابات الراتب المستحق دقيقة ومتوافقة مع البيانات المُدخلة.
- يعمل التطبيق بشكل سليم على متصفحات Chrome وFirefox وEdge الحديثة.
- واجهة المستخدم واضحة وسهلة الاستخدام بدون تدريب مسبق.
